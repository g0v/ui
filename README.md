# 如何參與？

# Issues 的 Labels 說明

- Designer 設計師提案
- Programer 工程師提案
- Architect 專案的架構討論
- Issue 討論元件的使用方法(ex. props)
- WIP 設計師/工程師 正在on going

# step
1. 提案 (create issue)

  - 畫Wireframe, 說明介面流程
  - 討論元件名稱

2. 規劃 (react api)

  - 列出有哪些 PropTypes
  - 建立元件的 file, 裡面的code可以都先不寫

3. 填坑 (等待沒有人 pull request)
